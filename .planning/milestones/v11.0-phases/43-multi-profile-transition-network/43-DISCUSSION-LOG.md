# DISCUSSION-LOG-43: Multi-Profile Transition (NETWORK)

## Discussion Context
- **Phase**: 43
- **Topic**: Enhancing professional networking by allowing role transitions from client accounts.

## Key Decisions

### 1. Verification & Access Control
- **Question**: Should role applications be auto-approved?
- **Decision**: **NO**. Panel access is blocked until the administrator validates the user's experience.
- **Rationale**: Maintain network quality and security.

### 2. Onboarding Experience
- **Question**: How to welcome new partners?
- **Decision**: Redirect to a specific **Welcome Landing Page** with an interactive **Guided Tour** (popups/tooltips).
- **Rationale**: Ensure partners understand the tools (Sidebar, Payouts, Events) immediately, reducing support overhead.

### 3. Role Switcher
- **Question**: Which roles to show in the switcher?
- **Decision**: **ALL** roles associated with the account.
- **Rationale**: Flexibility for power users who might be photographers and franchisees simultaneously.

### 4. Identity & Profile
- **Question**: What mandatory data should be added?
- **Decision**: **Profile Photo**.
- **Rationale**: Essential for professional identification and trust within the network.
