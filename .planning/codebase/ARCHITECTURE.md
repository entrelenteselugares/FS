# Architecture: Foto Segundo (Unified Phygital Ecosystem)

## Core Architectural Patterns

### 1. Unified Order Engine (O-Engine)

The system uses a centralized logic for handling all transaction types (B2B Upgrades, Marketplace Unit Sales, and Crowdfunded Events).

- **Controller**: `PaymentController` orchestrates the flow.
- **Service**: `PricingService` calculates dynamic pricing and complex commission splits.
- **Splits**: Implements a 4-tier distribution: Matriz (Plataforma), Captação (Vendedor/Franqueado), Edição (Editor), and Cartório (Legal).

### 2. IoT Printer Agent (Hardware Layer)

Decoupled automation for physical photo output.

- **Architecture**: Pull-based system. Local agent (IoT) polls `/api/admin/phygital/queue`.
- **Workflow**: `Poll -> Download -> Print Command (lp/powershell) -> Confirm`.
- **Resilience**: Temporary local storage and retry logic ensure prints are never lost due to network instability.

### 3. Passive Commission Model (B2B Expansion)

- **Hierarchy**: Professionals are linked to Partners (Franchisees) via `ProfessionalNetwork`.
- **Passive Split**: Every sale generated by a professional automatically yields a passive percentage to their host partner, calculated in real-time via `PricingService.calculateSplits`.

### 4. Phygital State Management

- **Real-time Unlocking**: Backend cross-references purchase history to dynamically populate `unlockedMediaIds`.
- **Hybrid Storage Pipeline**: 
    - **Hot Storage (Supabase)**: Real-time event media for immediate marketplace delivery.
    - **Cold Storage (Google Drive)**: High-persistence storage for "Memory Vaults", utilizing folder-per-album hierarchy.
    - **Proxy Relay**: Backend acts as a stream-proxy (`/vaults/media/proxy/:fileId`) to bypass Google Drive's restrictive CORS and authentication headers for frontend image rendering.

## Technology Stack

- **Frontend**: Vite + React + Tailwind + Framer Motion.
- **Backend**: Node.js + Express + Prisma + Supabase.
- **E2E Testing**: Playwright (penny-pix protocol).

## Key Patterns

- **Transactional Integrity**: Heavy use of `prisma.$transaction` for multi-step operations (e.g., Checkout -> Split -> Ledger).
- **Gamification Ledger**: Immutable record pattern for all loyalty points and credits.
- **Micro-Franchise Tiers**: State-machine logic for automatic tier progression based on volume.

## Decoupled IoT

The Printer Agent operates as a separate local entity, communicating with the core via standard API/WebSocket, ensuring the platform remains resilient to local connectivity issues.
